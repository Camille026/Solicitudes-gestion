# Instancia 2 — Desarrollo y debugging

## Objetivo

Corregir y completar el módulo en `codigo-base/` según tu **análisis de Instancia 1** y el comportamiento real del código legacy.

## Antes de tocar código

1. Tener completada [Plantilla-Analisis.md](../Instancia-1-Analisis/Plantilla-Analisis.md).
2. Completar el **diagnóstico de bugs** en [Plantilla-Entrega.md](../Plantilla-Entrega.md) **antes** de modificar archivos.

## Tareas

### Nivel mínimo

- [ ] El proyecto carga sin error fatal de PHP/JS.
- [ ] Listar, crear, editar y eliminar solicitudes.
- [ ] Campos obligatorios al crear/editar.
- [ ] Instrucciones de ejecución en la Plantilla de Entrega.

### Nivel estándar

- [ ] Reglas de edición, eliminación y cambio de estado coherentes con tu Instancia 1.
- [ ] Validaciones en código (no solo ocultar botones en pantalla).

### Nivel refinado

- [ ] Filtros y orden del listado según tu §7 de Instancia 1.
- [ ] Mensajes de error claros y comportamiento predecible.

## Restricciones

- **No** cambiar estructura de carpetas de `codigo-base/`.
- **No** cambiar contrato JSON `{ success, data | error }`.
- **No** renombrar `src/controllers/api.php`.
- Respetar supuestos documentados en Instancia 1.

## Entregables

Ver [Plantilla-Entrega.md](../Plantilla-Entrega.md): diagnóstico, código, log IA, instrucciones.
